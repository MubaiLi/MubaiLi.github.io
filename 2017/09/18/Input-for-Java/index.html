Java的输入（针对OJ）

发表于 2017-09-18

最近在刷姥姥的PAT乙级，感觉很不爽，各种超时。很无奈，Java的确不适合刷题，同一道题，这边Scanner还在启动，C++都运行完了。特别是PAT很多题目的限制都在100ms，感受到了对Java深深的恶意啊。。。

But，我们不能坐以待毙了，题还是要刷的。
那么，一个解决方案就是转移到牛客网，PAT乙级(Basic Level)真题。
额，然而有时候牛客网的OJ也超时，好尴尬。

辣么，最大的问题在哪儿呢？
Scanner is too Slooooow !!!

我们来回顾一下一般的Java的输入：

简洁明了，有木有，又好写，又好用。
可是慢呀。。。。
这时候我想起了BufferedReader，抛弃掉Scanner，转用BufferedReader，我惊奇的发现，之前好多超时的题目神奇的通过了，yeah~
以PAT乙级的1027. 打印沙漏 为例， 看一下利用BufferedReader来进行输入：

是不是感觉好麻烦，代码也不好写，感觉乱糟糟的，可是比Scanner快多了。

下面，我们来对比一下C还有Java的这两种输入方式：
同样都是输入一个double型的变量

  • C

  • Java with Scanner

  • Java with BufferedReader

然后是输入10,000,000个int型变量的时间对比：

Input method Time(sec)
C scanf(“%d”, &arg) 3.78
Scanner.parseInt() 29.52
BufferedReader + inline Integer.parseInt 2.89
BufferedReader + Reader.nextInt 3.01

真·没有对比就没有伤害。。

最后，会C/C++的就不要用Java刷题了。
